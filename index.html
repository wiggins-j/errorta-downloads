A run at a glance

  1. Pick the team. Assign each role to a model — a local model on Ollama, a Claude, Codex, or Cursor CLI subscription, or a provider API — and set what each one is allowed to see.
  2. Set the goal. Not just one instruction — hand it a whole outcome ("build an X that does A, B, and C"). The PM agent breaks it into a task backlog and runs it.
  3. You decide how models are chosen. Pin a model to a role, allow a family, or give the PM a shortlist and let it pick the cheapest capable model per task.
  4. Your gates hold. The run pauses where you asked — comment on the plan or approve a stage — or runs start to finish when you turn gates off.
  5. Review before merge. Work lands as branches and pull requests with real test runs. Nothing touches your tree until you review the diff and accept.

Coding is one mode. Errorta also convenes a Council of models for hard questions, and a Judge that grounds answers in your own data.

Goal & backlog
You give a plain-language goal. The PM agent decomposes it into tasks, assigns each to a role, and re-plans as work lands or stalls.
Model routing
Each task goes to the cheapest capable model you allow. The PM escalates to a stronger model only when a task needs it, and learns which models handle which kind of work, so assignments improve over time.
Context policy
Every member works inside a room where you set what it can see. Each context hand-off is auditable down to the byte, so a model only ever receives what its policy allows.
Governance gate
The run stops where you want it — review the plan, approve a stage — or let the team run start to finish. Governance is light, strict, or off.
Delivery
Branches, pull requests, and real test runs. The diff is yours to review; nothing is written to your working tree until you accept it.

How Errorta is different

A single coding agent — Claude Code, Cursor, Codex — takes a goal, works, and then stops. You re-prompt it for the next step, tell it to write tests, run them yourself, and keep its context from filling up with noise. You drive; it types.

Errorta wraps those same agents in a harness. You give a high-level goal and the PM runs the whole loop: it breaks the work down, assigns each task to a fresh agent with just enough context to succeed, has the dev build, the tester run your real tests, and the reviewer check the diff — merging and re-planning until the goal is met, pausing only at the gates you set. Same models, with a team and a manager around them.

You set the direction; the team does the driving. A high-level goal in, reviewed and tested code out — and nothing lands in your codebase until you accept it.

One team, the models you already have Alpha

A single run can mix subscriptions, local models, and provider APIs. A concrete team might read:

PM
Claude CLI — plans, assigns, holds the merge gate
Dev
Codex CLI — writes the code for each task
Tester
Ollama · Qwen — runs your real test commands; the verdict is the exit code
Reviewer
Cursor CLI — reviews the diff before merge

Also connects: Anthropic, OpenAI, and Google APIs, and any Ollama model such as Gemma or Mistral.

The Council Alpha

Not every question is a coding task. Convene a Council — several models deliberating in a shared room across a topology you choose, such as parallel answers, round-robin, debate, or blind review. Each member sees only what its policy allows, a finalizer or judge resolves the room, and every context hand-off is inspectable down to the byte. It is a second-opinion engine that runs on your terms.

Grounded, and honest about it Alpha

Errorta is built in tandem with AIAR, the open-source local-RAG framework that provides retrieval, LLM-as-judge evaluation, and a grounding and correction loop. Build a corpus from a plain-markdown brief, drag in your own documents, or point Errorta at a folder to watch. Your agents and the Council answer from that corpus with citations and a structured verdict on whether the answer holds up — and when you accept a correction, it carries into later runs. Local AI that admits when it is wrong and remembers what you fixed.
